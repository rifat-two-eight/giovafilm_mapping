Next.js API Integration Guide: Map & Place Modules
This guide details how to integrate the Map and Place modules from the backend into a Next.js frontend application.

1. Environment Setup
Add your backend API URL to your .env.local file in Next.js.

env
NEXT_PUBLIC_API_URL=http://10.10.7.50:4009/api/v1
2. Data Models (Types)
Match these interfaces in your Frontend to ensure type safety.

typescript
// src/types/index.ts
export type Status = 'Draft' | 'Published';

3. API Service Layer (Axios)
It's recommended to use a central Axios instance to handle common logic like Base URL and Authorization headers.

typescript
// src/services/api.ts
import axios from 'axios';
const api = axios.create({
  baseURL: process.env.NEXT_PUBLIC_API_URL,
});
// Interceptor to add Auth Token
api.interceptors.request.use((config) => {
  const token = typeof window !== 'undefined' ? localStorage.getItem('token') : null;
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});
export default api;
3. Map Module Integration
Endpoints
Action	Method	Endpoint	Auth
Get All Maps	GET	/maps	Public
Get Map Details	GET	/maps/:id	Public
Purchase Map	POST	/maps/:id/purchase	Restricted (User)
Get Purchased Maps	GET	/maps/purchased/all	Restricted (User)
API Functions
typescript
// src/services/mapService.ts
import api from './api';
export const mapService = {
  getAllMaps: (params?: any) => api.get('/maps', { params }),
  getMapById: (id: string) => api.get(`/maps/${id}`),
  purchaseMap: (id: string) => api.post(`/maps/${id}/purchase`),
  getPurchasedMaps: () => api.get('/maps/purchased/all'),
};

5. Usage in Next.js (Example)
Local Data Fetching (Client Component)
tsx
'use client';
import { useEffect, useState } from 'react';
import { mapService } from '@/services/mapService';
export default function MapsPage() {
  const [maps, setMaps] = useState([]);
  useEffect(() => {
    mapService.getAllMaps().then(res => setMaps(res.data.data));
  }, []);
  return (
    <div>
      <h1>Road Trip Maps</h1>
      {maps.map((map: any) => (
        <div key={map.id}>{map.name}</div>
      ))}
    </div>
  );
}
Server-Side Data Fetching (App Router)
tsx
// src/app/maps/[id]/page.tsx
import { mapService } from '@/services/mapService';
export default async function MapDetails({ params }: { params: { id: string } }) {
  const response = await mapService.getMapById(params.id);
  const map = response.data.data;
  return (
    <div>
      <h1>{map.name}</h1>
      <p>{map.description}</p>
    </div>
  );
}
6. Form Submission (Admin)
For endpoints requiring file uploads (Images/Media), use FormData.

typescript
const handleCreateMap = async (formData: any) => {
  const data = new FormData();
  data.append('name', formData.name);
  data.append('description', formData.description);
  
  // Multiple files
  formData.images.forEach((file: File) => {
    data.append('images', file);
  });
  await api.post('/maps', data, {
    headers: { 'Content-Type': 'multipart/form-data' },
  });
};
7. Redux Toolkit (RTK) Query Integration
Since Redux is already set up, you can use RTK Query for efficient data fetching and caching.

Create Base API
typescript
// src/redux/api/baseApi.ts
import { createApi, fetchBaseQuery } from '@reduxjs/toolkit/query/react';
export const baseApi = createApi({
  reducerPath: 'api',
  baseQuery: fetchBaseQuery({ 
    baseUrl: process.env.NEXT_PUBLIC_API_URL,
    prepareHeaders: (headers) => {
      const token = localStorage.getItem('token');
      if (token) {
        headers.set('authorization', `Bearer ${token}`);
      }
      return headers;
    },
  }),
  tagTypes: ['Map', 'Place'],
  endpoints: () => ({}),
});
Map API Endpoints
typescript
// src/redux/api/mapApi.ts
import { baseApi } from './baseApi';
export const mapApi = baseApi.injectEndpoints({
  endpoints: (build) => ({
    getMaps: build.query({
      query: (arg) => ({ url: '/maps', method: 'GET', params: arg }),
      providesTags: ['Map'],
    }),
    getMapById: build.query({
      query: (id) => ({ url: `/maps/${id}`, method: 'GET' }),
      providesTags: ['Map'],
    }),
    purchaseMap: build.mutation({
      query: (id) => ({ url: `/maps/${id}/purchase`, method: 'POST' }),
      invalidatesTags: ['Map'],
    }),
  }),
});
export const { useGetMapsQuery, useGetMapByIdQuery, usePurchaseMapMutation } = mapApi;
Place API Endpoints
typescript
// src/redux/api/placeApi.ts
import { baseApi } from './baseApi';
export const placeApi = baseApi.injectEndpoints({
  endpoints: (build) => ({
    getPlaces: build.query({

export const { useGetPlacesQuery, useGetPlaceByIdQuery } = placeApi;
Usage in Components
tsx
'use client';
import { useGetMapsQuery } from '@/redux/api/mapApi';
export default function MapsList() {
  const { data, isLoading } = useGetMapsQuery({});
  if (isLoading) return <div>Loading...</div>;
  return (
    <ul>
      {data?.data?.map((map: any) => (
        <li key={map.id}>{map.name}</li>
      ))}
    </ul>
  );
}
8. Design & Implementation Tips (API Based)
Based on the API structure, here are some design considerations for your UI/UX:

A. Geolocation handling
The API follows GeoJSON format: location: { coordinates: [longitude, latitude] }.
Note: Most maps (like Google/Leaflet) use [latitude, longitude]. Remember to swap them in your frontend logic when displaying on a map.
B. Multiple Media rendering
Both Map and Place models use arrays for images (images: string[] and media: string[]).
Design Update: Instead of a single image, use a Carousel or a Grid Gallery for the details page.
C. Status & Visibility
The status field is an enum: 'Draft' | 'Published'.
Ensure your frontend filters out Draft items for general users, or show a 'Saved' badge for Admins.
D. Pricing & Access
Use the isPaid boolean and price field to show price tags.
For isPaid: true, show a "Purchase to unlock" button redirecting to the /purchase mutation.
E. Dynamic Lists
Fields like features (Map) and services (Place) are arrays of strings.
Use Chips/Badges to display these instead of plain text for a better look.
TIP

Make sure the backend allows CORS for your Next.js frontend URL (e.g., http://localhost:3000).

IMPORTANT

Some endpoints require Admin privileges. Ensure the logged-in user has the appropriate role saved in their JWT token
