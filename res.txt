API Specification: Update Profile Image Only
অন্যান্য টেক্সট ফিল্ড ছাড়া শুধুমাত্র প্রোফাইল পিকচার আপলোড বা চেঞ্জ করার এপিআই রিকোয়েস্ট স্পেসিফিকেশন।
১. জেনারেল ইনফরমেশন (General Info)
HTTP Method: PATCH
Request URL: /api/v1/user/profile
Request Content-Type: multipart/form-data
Authentication: Bearer Token (প্রয়োজনীয়)
২. হেডার কনফিগারেশন (Request Headers)
Header Key	Header Value	Description
Authorization	Bearer <JWT_TOKEN>	ব্যবহারকারীর অ্যাক্সেস টোকেন
Content-Type	multipart/form-data	মাল্টিপার্ট ফরম্যাট (ফাইল আপলোডের জন্য)
৩. রিকোয়েস্ট বডি (Request Body - Form Data)
শুধুমাত্র একটি ফাইল ফিল্ড পাঠাতে হবে:
Parameter Key	Type	Value / File Type	Required	Description
images	File (Blob)	.jpeg, .jpg, .png (সর্বোচ্চ 10MB)	Yes	ব্যবহারকারীর নতুন প্রোফাইল ইমেজ ফাইল।
IMPORTANT
ফর্ম-ডাটার কী (Key) নামটি অবশ্যই images হতে হবে। ব্যাকএন্ডে Multer শুধুমাত্র এই নির্দিষ্ট কী-এর মাধ্যমেই প্রোফাইল পিকচার রিসিভ করে।
৪. রেসপন্স ফরম্যাট (Expected Responses)
ক. সফল রেসপন্স (Success Response - 200 OK)
ছবিটি সার্ভারে আপলোড ও অপ্টিমাইজ হওয়ার পর ডাটাবেজে প্রোফাইল পাথ আপডেট হয়ে যাবে এবং নিচের রেসপন্সটি আসবে:
json{  "statusCode": 200,  "success": true,  "message": "Profile updated successfully",  "data": "Profile updated successfully."}
খ. ব্যর্থ রেসপন্স (Error Response)
ফাইল ফরম্যাট ভুল হলে (400 Bad Request):
টোকেন না দিলে বা ইনভ্যালিড হলে (401 Unauthorized):
json{  "success": false,  "message": "You are not authorized"}
json{  "success": false,  "message": "Invalid file type for images"}
