Frontend Integration Guide: Google Maps Coordinate Extraction API
This document details how the frontend should communicate with the newly added backend coordinate extraction endpoint to resolve Google Maps links (both short and long links) and extract coordinates.
1. API Specifications
Endpoint: /api/v1/place/extract-coordinates
Method: POST
Headers:
Content-Type: application/json
Authorization: Bearer <ADMIN_OR_EDITOR_TOKEN> (Must be logged in as an Admin, Super Admin, or Map Editor)
Request Body
json{  "url": "https://maps.app.goo.gl/wJkZ4tPqZ4y2e8L47"}
Success Response (200 OK)
json{  "success": true,  "statusCode": 200,  "message": "Coordinates extracted successfully",  "data": {    "lat": 23.7808875,    "lng": 90.2680875  }}
Error Response (400 Bad Request)
If the URL cannot be resolved or does not contain coordinates:
json{  "success": false,  "message": "Could not extract coordinates. Try using the full URL from your browser address bar.",  "errorMessages": [ ... ]}
2. Recommended Frontend Workflow (React / Next.js)
To implement this before the user clicks the "Add Place" button:
Provide a Google Maps Link input field in the Form.
In the onChange or onBlur (when focus leaves the input) event, check if the input is a valid URL.
Show a loading spinner/indicator near the coordinates fields.
Send the POST request to the backend.
On success:
Extract lat and lng from response.data.data.
Update your form's coordinates state (e.g. latitude and longitude fields).
Pre-populate these values in the form or automatically place a marker on your interactive frontend map.
On error:
Alert the user using the error message returned by the server.
