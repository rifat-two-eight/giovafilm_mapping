# 🗺️ Map Editor Access Control — Frontend Developer Guide
 
This guide explains how to integrate the **Map Editor** feature from the frontend (Admin Dashboard).
 
---
 
## 🧭 Overview
 
The **Super Admin** can:

1. Invite someone as a **Map Editor** and assign maps/countries right away

2. Change an existing user's role to **Map Editor** and assign access

3. Update a Map Editor's assigned maps/countries at any time
 
A **Map Editor** can ONLY edit:

- Maps they are assigned to

- Places that belong to their assigned maps/countries

- Offers linked to those places
 
---
 
## 🔑 Authentication
 
All admin APIs require a **Bearer Token** in the header:
 
```

Authorization: Bearer <super_admin_jwt_token>

```
 
---
 
## 📌 API Endpoints
 
### 1. ✉️ Invite a New Map Editor (with assignment)
 
**Endpoint:**

```

POST /api/v1/users/invite

```
 
**Headers:**

```json

{

  "Authorization": "Bearer <super_admin_token>",

  "Content-Type": "application/json"

}

```
 
**Request Body:**

```json

{

  "email": "mapeditor@example.com",

  "role": "MAP_EDITOR",

  "assignedMaps": ["64f1a2b3c4d5e6f7a8b9c0d1", "64f1a2b3c4d5e6f7a8b9c0d2"],

  "assignedCountries": ["USA", "France"]

}

```
 
> ⚠️ `assignedMaps` = Array of **Map IDs** (MongoDB ObjectId strings)
> ⚠️ `assignedCountries` = Array of **country name strings** (e.g. "USA", "France")
 
**Response:**

```json

{

  "success": true,

  "statusCode": 200,

  "message": "User invited successfully."

}

```
 
> ✅ The server will automatically send an **invitation email** with an OTP to the user. No extra step needed from the frontend for email.
 
---
 
### 2. 🔄 Change Existing User's Role to Map Editor (with assignment)
 
**Endpoint:**

```

PATCH /api/v1/users/update-role/:userId

```
 
**Headers:**

```json

{

  "Authorization": "Bearer <super_admin_token>",

  "Content-Type": "application/json"

}

```
 
**Request Body:**

```json

{

  "role": "MAP_EDITOR",

  "assignedMaps": ["64f1a2b3c4d5e6f7a8b9c0d1"],

  "assignedCountries": ["Spain"]

}

```
 
**Response:**

```json

{

  "success": true,

  "statusCode": 200,

  "message": "User role updated successfully."

}

```
 
---
 
### 3. 🗺️ Update Assigned Maps/Countries for an Existing Map Editor
 
Use this when the Super Admin wants to **change access** for a Map Editor who is already set up (without changing their role).
 
**Endpoint:**

```

PATCH /api/v1/users/assign-editor-access/:userId

```
 
**Headers:**

```json

{

  "Authorization": "Bearer <super_admin_token>",

  "Content-Type": "application/json"

}

```
 
**Request Body:**

```json

{

  "assignedMaps": ["64f1a2b3c4d5e6f7a8b9c0d3"],

  "assignedCountries": ["Italy", "Germany"]

}

```
 
**Response:**

```json

{

  "success": true,

  "statusCode": 200,

  "message": "Editor access updated successfully."

}

```
 
> ✅ Both `assignedMaps` and `assignedCountries` are optional. You can send only one if needed.
 
---
 
## 📋 Where to Get Map IDs?
 
Call the Get All Maps API to populate the multi-select dropdown:
 
**Endpoint:**

```

GET /api/v1/maps

```
 
**Response:**

```json

{

  "data": [

    { "_id": "64f1a2b3c4d5e6f7a8b9c0d1", "name": "Paris Map", "country": "France" },

    { "_id": "64f1a2b3c4d5e6f7a8b9c0d2", "name": "New York Map", "country": "USA" }

  ]

}

```
 
Use `_id` as the value and `name` as the label in the dropdown.

## 🖥️ Suggested UI Flow (Admin Dashboard)
 
### Invite Map Editor Page

```

[ Email Input ]

[ Role Dropdown ] → Select: Map Editor

    ↓ (when MAP_EDITOR is selected, show extra fields)

[ Assigned Maps ] → Multi-select dropdown (fetched from GET /maps)

[ Assigned Countries ] → Multi-select or text input (e.g. "USA", "France")

[ Send Invite Button ]

```
 
### Edit User / Change Role Page

```

[ Role Dropdown ]

    ↓ (when MAP_EDITOR is selected, show extra fields)

[ Assigned Maps ] → Multi-select dropdown

[ Assigned Countries ] → Multi-select or text input

[ Save Button ]

```
 
### User Detail / Editor Settings Page

```

[ Current Assigned Maps: Paris Map, NY Map ]

[ Current Assigned Countries: USA, France ]

[ Edit Access Button ]

    ↓

[ Assigned Maps ] → Multi-select

[ Assigned Countries ] → Text tags input

[ Update Button ] → PATCH /assign-editor-access/:userId

```

## ⚠️ Error Responses to Handle
 
| Status Code | Meaning | When it happens |

|---|---|---|

| `400` | Bad Request | Missing required fields |

| `401` | Unauthorized | Token missing or expired |

| `403` | Forbidden | User not authorized for that map |

| `404` | Not Found | User or Map not found |
 
---
 
## 🔄 Role Values Reference
 
| Role | Value to send in API |

|---|---|

| Super Admin | `SUPER_ADMIN` |

| Admin | `ADMIN` |

| Map Editor | `MAP_EDITOR` |

| User | `USER` |
 
---
 
## ✅ Quick Checklist for Frontend Dev
 
- [ ] Fetch map list from `GET /api/v1/maps` for dropdown

- [ ] Show `assignedMaps` and `assignedCountries` fields only when role = `MAP_EDITOR`

- [ ] Use Map `_id` values (not names) when sending `assignedMaps`

- [ ] Use country name strings (e.g. `"USA"`) when sending `assignedCountries`

- [ ] Handle `403` errors gracefully when a Map Editor tries to edit outside their scope

- [ ] Show current assigned maps/countries on the Map Editor's profile/detail page
