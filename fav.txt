Next.js প্রজেক্টে favourite মডিউলটি ইন্টিগ্রেট করার জন্য একটি পূর্ণাঙ্গ গাইড নিচে দেওয়া হলো। এখানে টাইপ ডেফিনিশন থেকে শুরু করে সার্ভার এবং ক্লায়েন্ট সাইড ব্যবহারের উদাহরণ দেওয়া হয়েছে।
 
### ১. টাইপ এবং ইন্টারফেস তৈরি (Types & Interfaces)
প্রথমে আপনার Next.js প্রজেক্টের types ফোল্ডারে একটি ফাইল তৈরি করুন (যেমন: favourite.ts ):
 
```
// types/favourite.ts
 
export type FavouriteType = 'Map' | 
'Place';
 
export interface IFavourite {
  _id: string;
  user: string;
  map?: string;
  place?: string;
  type: FavouriteType;
  createdAt: string;
  updatedAt: string;
}
 
export interface ApiResponse<T> {
  success: boolean;
  statusCode: number;
  message: string;
  meta?: {
    page: number;
    limit: number;
    total: number;
  };
  data: T;
}
```
### ২. API সার্ভিস তৈরি (API Service)
API কল করার জন্য একটি সার্ভিস ফাইল তৈরি করুন। এখানে fetch ব্যবহার করা হয়েছে:
 
```
// services/favouriteService.ts
 
const BASE_URL = process.env.
NEXT_PUBLIC_API_URL;
 
export const favouriteService = {
  // ১. ফেভারিট লিস্ট গেট করা
  async getMyFavourites(token: 
  string, query: string = '') {
    const res = await fetch(`$
    {BASE_URL}/favourite?${query}`, 
    {
      headers: {
        Authorization: `Bearer $
        {token}`,
      },
    });
    return res.json();
  },
 
  // ২. ফেভারিট টগল করা (Add/Remove)
  async toggleFavourite(token: 
  string, data: { type: 
  FavouriteType; map?: string; 
  place?: string }) {
    const res = await fetch(`$
    {BASE_URL}/favourite`, {
      method: 'POST',
      headers: {
        'Content-Type': 
        'application/json',
        Authorization: `Bearer $
        {token}`,
      },
      body: JSON.stringify(data),
    });
    return res.json();
  },
 
  // ৩. নির্দিষ্ট ফেভারিট রিমুভ করা
  async removeFavourite(token: 
  string, id: string) {
    const res = await fetch(`$
    {BASE_URL}/favourite/${id}`, {
      method: 'DELETE',
      headers: {
        Authorization: `Bearer $
        {token}`,
      },
    });
    return res.json();
  },
};
```
### ৩. ক্লায়েন্ট সাইড কম্পোনেন্টে ব্যবহার (Client Side Usage)
React Hook (যেমন: useState , useEffect ) বা React Query ব্যবহার করে আপনি এটি ইন্টিগ্রেট করতে পারেন:
 
```
'use client';
import { useState } from 'react';
import { favouriteService } from '@/
services/favouriteService';
 
export default function 
FavouriteButton({ itemId, type }: { 
itemId: string, type: 'Map' | 
'Place' }) {
  const [loading, setLoading] = 
  useState(false);
  const token = 
  'YOUR_AUTH_TOKEN'; // এটি আপনার 
  কনটেক্সট বা কুকি থেকে আসবে
 
  const handleToggle = async () => {
    setLoading(true);
    try {
      const payload = type === 
      'Map' ? { type, map: itemId } 
      : { type, place: itemId };
      const response = await 
      favouriteService.
      toggleFavourite(token, 
      payload);
      if (response.success) {
        alert(response.message);
      }
    } catch (error) {
      console.error('Error toggling 
      favourite:', error);
    } finally {
      setLoading(false);
    }
  };
 
  return (
<button onClick={handleToggle} 
    disabled={loading}>
      {loading ? 'Processing...' : 
      'Toggle Favourite'}
</button>
  );
}
```
### ৪. সার্ভার সাইড ফেচিং (Next.js Server Component)
যদি আপনি সার্ভার থেকে ডেটা লোড করতে চান:
 
```
// app/dashboard/favourites/page.tsx
import { favouriteService } from '@/
services/favouriteService';
import { cookies } from 'next/
headers';
 
export default async function 
FavouritesPage() {
  const cookieStore = cookies();
  const token = cookieStore.get
  ('accessToken')?.value;
  const response = await 
  favouriteService.getMyFavourites
  (token || '');
  const favourites = response.
  data || [];
 
  return (
<div>
<h1>My Favourites</h1>
<ul>
        {favourites.map((item: any) 
        => (
<li key={item._id}>
            {item.type}: {item.
            map || item.place}
</li>
        ))}
</ul>
</div>
  );
}
```
### গুরুত্বপূর্ণ নোটসমূহ:
- Authentication : favourite মডিউলের সব এন্ডপয়েন্টে auth মিডলওয়্যার প্রয়োজন। তাই হেডার-এ অবশ্যই Authorization: Bearer <token> পাঠাতে হবে।
- Validation : POST মেথডে map অথবা place যেকোনো একটি পাঠাতে হবে (দুটো একসাথে পাঠানো যাবে না), যা আপনার favourite.validation.ts এ ডিফাইন করা আছে।
- Error Handling : API থেকে আসা success প্রপার্টি চেক করে ইউজারকে ফিডব্যাক দেওয়া উচিত।
